# 프로젝트명: 마이링크 (My Link) - 기능 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트 목적**: 여러 소셜 미디어, 포트폴리오, 웹사이트 등의 링크를 단 하나의 고유 URL로 모아서 공유할 수 있게 해주는 개인화된 멀티 링크 프로필 서비스 (Linktree 클론).
- **대상 사용자**:
  - 다수의 소셜 미디어 및 운영 사이트를 하나의 프로필에서 홍보하고 싶은 크리에이터, 인플루언서
  - 포트폴리오 및 운영 채널을 모아서 보여주고 싶은 프리랜서 및 아티스트
  - 복잡한 웹사이트 제작 없이 간단한 개인 소개 페이지가 필요한 일반 사용자
- **서비스 형태**: 웹 전용 서비스 (반응형 웹 지원)
- **수익 모델**: 100% 무료 제공 (부분 유료화 없음)

## 2. 핵심 기능 목록 (우선순위)

### 2.1 필수 핵심 기능 (v1.0 런칭 목표)
- **회원가입 및 로그인**: 파이어베이스(Firebase) 기반 구글 소셜 로그인 전용
- **사용자 온보딩 및 식별자 할당**: 가입 시 구글 지메일 아이디의 앞부분을 파싱하여 기본 닉네임(`displayName`)으로 설정하며, 이를 고유 프로필 주소의 URL 슬러그(Slug)로 사용
- **계정 및 프로필 관리**: 
  - 프로필 이미지 설정 및 추후 수정 가능
  - 프로필 닉네임 및 한 줄 소개(Bio) 설정 및 추후 수정 가능
- **링크 블록 관리**: 
  - 링크 추가, 수정, 삭제 (순서 변경 기능 없음)
  - 링크 추가 시 구글 파비콘 API(Google Favicon API)를 사용하여 해당 링크의 파비콘을 기본 아이콘으로 설정
  - 특정 링크 노출 상태(ON/OFF) 전환 기능 
- **공유용 웹 프로필 제공**: 사용자가 설정한 정보를 바탕으로 외부인에게 제공되는 방문자용 반응형 고유 웹페이지 랜딩 (`/displayName`)

### 2.2 향후 업데이트 및 선택적 확장 기능 (v2.0 이후)
- [ ] 상세 방문 통계 및 전환율(Analytics) 대시보드
- [ ] SEO 설정 및 Open Graph (소셜 미디어 공유 썸네일) 설정 기능
- [ ] 단순 텍스트 링크 외에 서드파티 미디어 임베드 (유튜브, 스포티파이 등)
- [ ] 특정 기간 한정 링크 예약(스케줄링) 노출 설정 기능
- [ ] 명함 및 오프라인 홍보용 사용자 전용 QR 코드 자동 생성 기능 

## 3. 각 기능의 상세 설명 (v1.0 필수 기능 기준)

### 3.1. 인증 및 사용자 관리
- **Firebase 소셜 로그인**
  - 구글 계정 정보를 이용하여 별도의 비밀번호 찾기나 이메일 인증 절차 없이, 보안성 높고 빠른 원 클릭 로그인 동작.
  - 로그인 직후 사용자의 세션을 유지하며 대시보드 관리자 단으로 렌더링.

### 3.2. 고유 주소 및 프로필
- **고유 식별자 처리**
  - 구글 소셜 로그인 후 이메일(Gmail) 주소의 맨 앞부분(@ 앞부분)을 추출하여 기본 프로필 닉네임(`displayName`)으로 자동 설정.
  - 회원별로 제공되는 웹페이지 주소(URL)는 이 `displayName`을 슬러그로 활용하여 `domain.com/displayName` 형태를 제공.
- **프로필 기본 정보**
  - 방문자가 프로필 접속 시 상단 중앙에 노출되는 가장 중요한 아이덴티티 시각화 영역. 이미지 최적화 및 텍스트 글자 수 제한 등의 벨리데이션(Validation) 적용.

### 3.3. 링크 관리
- **실시간 링크 편집 인터페이스**
  - **추가/수정**: 링크를 보여줄 제목(Title)과 목적지(URL 주소)를 매핑하여 관리 리스트에 추가 (구글 파비콘 API를 사용하여 아이콘 자동 설정). 기존 링크의 내용을 수정할 때는 별도 페이지 이동 없이 **인라인 편집(Inline Edit)** 방식을 사용함. 순서 변경은 지원하지 않음.
  - **온/오프 토글**: 영구적인 삭제 대신, 일시적인 사용 중단을 원할 때 활성화 스위치(Toggle)를 통해 클릭 한 번으로 방문자 화면에서 노출 또는 숨김 처리.

### 3.4. 데이터베이스 모델링
- **사용자 프로필 및 서브 컬렉션 구조**
  - `Users` 컬렉션: 사용자 기본 정보 저장
    ```json
    {
      "uid": "string (고유 식별자)",
      "email": "string",
      "displayName": "string (프로필 닉네임 겸 고유 주소 URL 슬러그, 초기값은 Gmail ID 앞부분)",
      "bio": "string (한 줄 소개)",
      "profileImageUrl": "string (프로필 이미지 URL)",
      "createdAt": "timestamp",
      "updatedAt": "timestamp"
    }
    ```
  - `Links` 서브 컬렉션: `Users`의 하위 서브 컬렉션으로 구성되어 생성된 링크 정보를 별도 분리하여 관리
    ```json
    {
      "id": "string (문서 고유 ID)",
      "title": "string (링크 제목)",
      "url": "string (목적지 URL)",
      "faviconUrl": "string (구글 API를 통해 추출된 파비콘 URL)",
      "isVisible": "boolean (노출 상태 ON/OFF)",
      "createdAt": "timestamp",
      "updatedAt": "timestamp"
    }
    ```

### 3.5. 퍼블릭 프로필 페이지 (방문자 브라우저 뷰)
- **반응형 렌더링 시스템**
  - 모바일(스마트폰) 세로 화면 비율에 가장 최적화되어 제작되며, 태블릿/PC 환경에서 접속 시에도 가운데 정렬 등의 처리를 통해 심미적으로 안정감 있는 뷰 제공.
